Angular 2.0 Components

Angular 2 applications contain building blocks called components. An application is essentially a tree of components, all stemming off of the root component.

Open the project in VS. You made a directory earlier named allergen. Within the allergen folder, we have cloned the github repository named ‘Demo’. From there we are going to start our focus on the folder named app.

The root component of our application is under src/app/app.component.ts. If you open it up, it contains the template, which is used to render the view in the application. The template contains HTML code, binding and directives. It also contains the selector, which tells the application where in the DOM the component should appear. 

@Component({
                selector: 'my-app',
                template: `<code><h1>Hello {{name}}</h1></code>`,
                })
The app.component.ts file also includes a class decorator. A class decorator includes the class name, the property name, the property type, and the value. 

export class AppComponent  { name = 'Angular'; }
The class name is AppComponent. The export keyword is used so that this component can be used in other modules. The property type is name and the value is ‘Angular’.

We could give the property a name by changing the HTML within the braces of export class to {appTitle: string = ‘AllerGenius’;} When you save this, it will be automatically updated in the browser.

Building the Web Application with Angular 2.0

Now that we know the basics and how to set up our Angular 2.0 web application, we can begin building the application. We are going to load our application through a new created HTML file. Create a new typescript file in the src file called app.component.html. In this file, paste the following: 

<h1>{{appTitle}}</h1>

In the app.component.ts file, change the template line of code to:

templateUrl: ‘app/app.component.html’,
This will update your page to include the html in the separate file as your template code. The website will then look like this:



Next create a new typescript component file named menuitem.component.ts. Add the following to your code:

import { Component } from '@angular/core';

    @Component({
      selector: 'menu',
      template: '<h2>Menu Items</h2>'
    })
    
    export class MenuItemComponent {
    }
To import the Menu component into our app component we need to open the app.module.ts file. In this file, add this code to the import section below the AppComponent import:

import { MenuItemComponent } from './menuitem.component';
Also, add the MenuItemComponent to the declarations section of the @NgModule.

@NgModule({
      imports: [BrowserModule],
      declarations: [AppComponent, MenuItemComponent],
      bootstrap:    [ AppComponent ]
    })
Make sure to add the ‘<menu></menu>’ selector to the original app.component.html file, so that the menu items component is added to the page. 

<h1>{{appTitle}}</h1>
        <menu></menu>

The goal of the next part of our application is to make it so when the allergens are updated, the menu items shown also update. First, we need to gather the updated allergens using the update button. This will be handled by the function we create that is attached to the change of the check boxes. Call this function selectionChange() and make its input an HTMLInputElement. It will return nothing.

The selectionChange() function will see if the checkbox is checked off. If it is checked off, the application will add that allergen to the list of allergens stored in the component class. If it is not checked off, the application will remove that allergen from the list. First, create the blank list of allergens and store it in the MenuItemComponent class.

allergensList: string[] = [];
Insert the selectionChange() function into the MenuItemComponent class as well: 

selectionChange(input: HTMLInputElement) {
      if (input.checked === true) {
       this.allergensList.push(input.id);
      } else {
        this.allergensList.splice(this.allergensList.indexOf(input.id), 1);
      }
     }

          this.pandaExpress = true;
      }
    
      }
Now that the allergen list is created, we need to dynamically change the food items shown based on the restaurant they belong to and if they contain the allergens selected by the user. We will do this using AngularJS’ ngIf directive. We are already displaying the objects using the ngFor directive, but we need to use ngIf to correctly filter these items.

First we need to filter the items by restaurant. The ngIf directive will use a function called checkRestaurant() and will return true if the boolean for the restaurant being passed through the function is the same restaurant that is selected. Add the following to your code in place of the ngFor function that displays the food items: 

<ng-container *ngIf="checkRestaurant('ChickFilA')">
        <p>Chick Fil A</p>
    </ng-container>
    
    <ng-container *ngIf="checkRestaurant('McDonalds')">
        <p>McDonalds</p>
    </ng-container>
    
    <ng-container *ngIf="checkRestaurant('PandaExpress')">
        <p>Panda Express</p>
    </ng-container>
Go back into the menuitem.component.ts file. We need to create the checkRestaurant() function, which will have an input string of the restaurant name. Based on the restaurant input, if the restaurant name matches the corresponding boolean with the value of true, the function will return true and display the items for that restaurant. 

checkRestaurant(restaurant: string): boolean{

        if (restaurant == "ChickFilA" && this.chickFilA === true) {
          return true;
        } else if (restaurant == "McDonalds" && this.mcDonalds === true) {
          return true;
        } else if (restaurant == "PandaExpress" && this.pandaExpress === true) {
          return true;
        }else{
          return false;
        }
    }

The last function we need to create to finish our application is the checkAllergens() function. The input for this function is the index of the menu item being accessed in the for loop and the name of the restaurant, so that the function can search through the correct menu. This function returns a true or false value in order to let the program know if the item should be displayed based on the allergens it contains. It loops through each allergen in our list and checks the item’s allergens for a match. If there isn’t a match, it returns true. 

checkAllergens(i: number, restaurant: string): boolean {

            if (restaurant == "ChickFilA") {
              for (let item of this.allergensList) {
                if (this.chickFilAMenu[i].allergens.includes(item)) {
                  return false;
                }
              }
            } else if (restaurant == "McDonalds") {
              for (let item of this.allergensList) {
                if (this.mcDonaldsMenu[i].allergens.includes(item)) {
                  return false;
                }
              }
            } else if (restaurant == "PandaExpress") {
              for (let item of this.allergensList) {
                if (this.pandaExpressMenu[i].allergens.includes(item)) {
                  return false;
                }
              }
            }
        
            return true;
        
          }

We will be adding a field for first name, last name, email, gender, opt-in for email updates, where you heard about us, allergens we should add, and restaurants we should add. If you want the results sent to your email, be sure to update the form's post settings.
